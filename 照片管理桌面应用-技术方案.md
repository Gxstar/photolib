# 照片管理桌面应用 — 需求分析与技术方案

> 目标：打造一款轻量、高效、面向专业摄影师的本地照片管理工具，对标 Adobe Bridge 的核心能力但去掉臃肿部分。

---

## 一、需求梳理

### 你提出的核心需求

| # | 需求 | 说明 |
|---|------|------|
| 1 | **存储卡导入** | 从相机 SD/CF 卡等可移动介质导入照片到本地磁盘 |
| 2 | **智能重命名** | 导入时按规则自动重命名，支持日期、相机型号、序号等变量 |
| 3 | **文件夹浏览** | 浏览整机或指定目录的照片，支持缩略图网格和单张预览 |
| 4 | **EXIF 信息展示** | 读取并展示详细 EXIF 元数据（需 exiftool 支持） |
| 5 | **EXIF 筛选** | 根据 EXIF 字段（相机型号、镜头、光圈、ISO、焦距等）过滤照片 |

### 我补充的建议

| # | 需求 | 理由 |
|---|------|------|
| 6 | **本地目录数据库** | 首次扫描建立 SQLite 目录，后续秒开，避免每次都全盘扫描 |
| 7 | **RAW 全格式支持** | 专业摄影师大量使用 RAW（CR3/NEF/ARW/DNG 等），exiftool 天然支持 |
| 8 | **评分与标记** | 星级评分 + 色标 + 旗标（Pick/Reject），这是 Bridge 的核心工作流 |
| 9 | **缩略图缓存系统** | 独立缓存层，大图浏览时秒加载，避免每次都渲染 RAW |
| 10 | **虚拟合集** | 不移动物理文件，基于条件或手动选择创建虚拟相册 |
| 11 | **批量操作** | 批量重命名、格式转换、尺寸调整、元数据修改 |
| 12 | **重复检测** | 基于文件哈希检测重复照片，清理存储空间 |
| 13 | **并排对比** | 两张照片并排放大对比，帮助选片 |
| 14 | **导出预设** | 可配置的导出参数（尺寸、格式、质量、水印、色彩空间） |
| 15 | **文件夹监控** | 监控指定目录，新增照片自动入库 |
| 16 | **RAW+JPEG 配对** | 识别同一张照片的 RAW 和 JPEG 版本，按需显示 |

## 二、技术选型

### 2.1 桌面框架

| 方案 | 包体积 | 性能 | 生态 | 难度 | 推荐 |
|------|--------|------|------|------|------|
| **Tauri v2** (Rust + WebView) | ~5-10 MB | ★★★★★ | ★★★☆ | 中等 | ⭐ **首选** |
| Electron (Chromium + Node.js) | ~150 MB | ★★★☆ | ★★★★★ | 较低 | 备选 |
| Wails (Go + WebView) | ~8 MB | ★★★★ | ★★☆ | 中等 | 备选 |

**推荐 Tauri v2**，理由：
- 极小的包体积，对比 Bridge 那种几百 MB 的体量是天壤之别
- Rust 后端性能优异，文件 I/O 和图像处理天然高效
- 调用 exiftool 等外部工具方便（Rust `Command`）
- 跨平台：Windows / macOS / Linux 一套代码

如果你的团队对 Rust 不熟，Electron 作为备选也完全可行，生态更成熟，尤其是 sharp 库处理图像很顺手。

### 2.4 EXIF 方案

**exiftool** (Perl, 由 Phil Harvey 维护) 是不二之选：

- 支持 **25,000+** 相机型号的 MakerNotes
- 覆盖几乎所有 RAW 格式
- 命令行调用，Rust/Node 端通过子进程交互
- 支持 JSON 输出（`-j`），解析方便
- 同时支持元数据写入

使用方式：
```bash
# 读取全部 EXIF 输出 JSON
exiftool -j -G -a photo.cr3

# 按需读取指定字段
exiftool -j -EXIF:DateTimeOriginal -EXIF:Make -EXIF:Model photo.cr3
```

**关键设计决策**：exiftool 启动有冷开销（~0.5-1s），因此应该：
1. 导入时一次性全字段提取，存入 SQLite，不要每张图都启动一次 exiftool
2. 批量模式下使用 `-stay_open` 管道模式，保持 exiftool 进程常驻
3. 只在需要「实时刷新元数据」时才重新调用

### 3.3 缩略图策略

三层缓存：

| 层级 | 尺寸 | 格式 | 用途 | 存储 |
|------|------|------|------|------|
| L1 微缩图 | 320px | JPEG Q=70 | 网格浏览 | SQLite BLOB / 独立目录 |
| L2 预览图 | 1920px | JPEG Q=85 | 单张预览 | `~/.photolib/thumbs/` |
| L3 全尺寸 | 原始 | 原始 | 1:1 放大查看 | 原始文件 |

生成策略：导入时同步生成 L1，后台队列生成 L2。

## 四、数据库设计

### 4.1 核心表

```sql
-- 照片主表
CREATE TABLE photos (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    file_path     TEXT NOT NULL UNIQUE,          -- 绝对路径
    file_name     TEXT NOT NULL,                 -- 文件名
    file_size     INTEGER,                       -- 文件大小 (bytes)
    file_hash     TEXT,                          -- SHA-256 文件哈希
    file_date     INTEGER,                       -- 文件修改时间
    media_type    TEXT,                          -- 'raw' | 'jpeg' | 'tiff' | 'heic' | etc.
    thumbnail_l1  BLOB,                          -- L1 缩略图
    thumbnail_l2  TEXT,                          -- L2 缩略图路径
    
    -- 核心 EXIF 字段（冗余存储，加速筛选）
    date_taken       TEXT,                       -- DateTimeOriginal
    camera_make      TEXT,                       -- Make
    camera_model     TEXT,                       -- Model
    lens_model       TEXT,                       -- LensModel
    focal_length     REAL,                       -- 焦距
    aperture         REAL,                       -- 光圈 (f-number)
    shutter_speed    TEXT,                       -- 快门速度 (如 "1/250")
    iso              INTEGER,                    -- ISO
    exposure_comp    REAL,                       -- 曝光补偿
    flash            INTEGER,                    -- 闪光灯
    white_balance    TEXT,                       -- 白平衡
    metering_mode    TEXT,                       -- 测光模式
    image_width      INTEGER,
    image_height     INTEGER,
    color_space      TEXT,                       -- sRGB / AdobeRGB / ProPhoto
    
    -- GPS
    latitude         REAL,
    longitude        REAL,
    altitude         REAL,
    
    -- 用户数据
    rating           INTEGER DEFAULT 0,          -- 0-5 星
    color_label      TEXT,                       -- Red/Blue/Green/Yellow/Purple
    flag             TEXT,                       -- 'pick' | 'reject' | NULL
    notes            TEXT,                       -- 备注
    
    -- 配对
    paired_raw_id    INTEGER,                    -- RAW 配对 (JPEG 指向 RAW)
    
    created_at       TEXT DEFAULT (datetime('now')),
    updated_at       TEXT DEFAULT (datetime('now'))
);

-- 索引（筛选性能关键）
CREATE INDEX idx_date_taken    ON photos(date_taken);
CREATE INDEX idx_camera_model  ON photos(camera_model);
CREATE INDEX idx_lens_model    ON photos(lens_model);
CREATE INDEX idx_focal_length  ON photos(focal_length);
CREATE INDEX idx_aperture      ON photos(aperture);
CREATE INDEX idx_iso           ON photos(iso);
CREATE INDEX idx_rating        ON photos(rating);
CREATE INDEX idx_flag          ON photos(flag);
CREATE INDEX idx_lat_lon       ON photos(latitude, longitude);
CREATE INDEX idx_file_hash     ON photos(file_hash);

-- 文件夹表
CREATE TABLE folders (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    path          TEXT NOT NULL UNIQUE,
    display_name  TEXT,
    photo_count   INTEGER DEFAULT 0,
    last_scan     TEXT,
    parent_id     INTEGER REFERENCES folders(id)
);

-- 合集表
CREATE TABLE collections (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    name          TEXT NOT NULL,
    description   TEXT,
    is_smart      INTEGER DEFAULT 0,            -- 是否智能合集
    filter_json   TEXT,                          -- 智能合集筛选条件 JSON
    created_at    TEXT DEFAULT (datetime('now'))
);

-- 合集-照片关联表
CREATE TABLE collection_photos (
    collection_id INTEGER REFERENCES collections(id) ON DELETE CASCADE,
    photo_id      INTEGER REFERENCES photos(id) ON DELETE CASCADE,
    sort_order    INTEGER,
    PRIMARY KEY (collection_id, photo_id)
);

-- 导入历史
CREATE TABLE imports (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    source_path   TEXT,                          -- 来源路径 (如 SD 卡)
    dest_folder   TEXT,                          -- 目标文件夹
    file_count    INTEGER,
    rename_rule   TEXT,                          -- 使用的重命名规则
    imported_at   TEXT DEFAULT (datetime('now'))
);
```

## 五、智能重命名系统

### 5.1 可用变量

| 变量 | 说明 | 示例 |
|------|------|------|
| `{date}` | 拍摄日期 | `2026-06-15` |
| `{time}` | 拍摄时间 | `143052` |
| `{datetime}` | 日期+时间 | `2026-06-15_143052` |
| `{camera_make}` | 相机制造商 | `Canon` |
| `{camera_model}` | 相机型号 | `EOS R5` |
| `{lens}` | 镜头型号简写 | `RF70-200` |
| `{focal}` | 焦距 | `85mm` |
| `{aperture}` | 光圈 | `f2.8` |
| `{iso}` | ISO | `ISO800` |
| `{seq:3}` | 三位序号 | `001`, `002` |
| `{seq:4}` | 四位序号 | `0001` |
| `{original}` | 原文件名（不含扩展名） | `IMG_1234` |
| `{ext}` | 扩展名 | `.CR3` |
| `{custom}` | 用户自定义文本 | `旅拍-日本` |

### 5.2 预设模板

- **日期型**: `{datetime}_{seq:3}`
  → `2026-06-15_143052_001.CR3`

- **机型前缀型**: `{camera_model}_{date}_{seq:4}`
  → `EOS_R5_2026-06-15_0001.CR3`

- **自定义前缀型**: `{custom}_{date}_{seq:3}`
  → `旅拍-日本_2026-06-15_001.CR3`

- **保留原名**: `{original}_{date}`
  → `IMG_1234_2026-06-15.CR3`

## 八、关键技术细节

### 8.1 支持的格式

**RAW 格式**（通过 exiftool）：
`.CR3` `.CR2` (Canon), `.NEF` `.NRW` (Nikon), `.ARW` `.SRF` `.SR2` (Sony), `.RAF` (Fujifilm), `.ORF` (Olympus), `.DNG` (通用), `.RW2` (Panasonic), `.PEF` (Pentax), `.3FR` (Hasselblad), `.IIQ` (Phase One)

**常规格式**：
`.JPEG` `.JPG` `.PNG` `.TIFF` `.TIF` `.HEIC` `.HEIF` `.WEBP` `.AVIF` `.BMP`

**视频**（可扩展）：
`.MP4` `.MOV` `.AVI`

### 8.2 性能考量

| 场景 | 策略 |
|------|------|
| 10万+ 照片目录 | SQLite + 索引 + 分页查询 |
| 缩略图网格 | 虚拟滚动，仅渲染可见区域 |
| RAW 缩略图生成 | 异步队列，线程池，优先级排序 |
| exiftool 批量读取 | `-stay_open` 管道模式，避免重复启动 |
| 文件夹扫描 | 增量扫描，对比 mtime |
| 前端渲染 | React.memo + useMemo，避免不必要重渲染 |

### 8.3 exiftool 集成要点

```rust
// Rust 端 - stay_open 管道模式示例思路
// 启动 exiftool 作为常驻进程
// 通过 stdin 发送文件名，stdout 读取 JSON 结果
```

### 8.4 跨平台注意事项

- 路径分隔符：Rust `PathBuf` 自动处理
- 可移动介质检测：Windows 用 `GetLogicalDrives`，macOS 用 `/Volumes`
- 文件系统大小写：macOS 默认不区分大小写，Windows 不区分，Linux 区分
- 缩略图缓存路径：使用 `dirs` crate 获取平台标准路径

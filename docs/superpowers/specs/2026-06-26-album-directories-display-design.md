# 相册侧边栏显示已添加目录

## 背景

当前"相册"tab 的左侧边栏是一个扁平列表，每个条目代表一个已注册的目录。当用户添加了多个目录后，侧边栏看起来是一堆平铺的目录行，缺少"这个相册 = 这些目录"的整体包装。

用户希望在相册 tab 下，先看到"总相册"这个相册实体，下面嵌套列出所有已添加的目录，再用缩进和可视化层级表达归属关系。

## 设计方案

### 选项

**方案 A（选中）**：纯 UI 改造。后端零改动。把现有目录列表在视觉上包一层"总相册"标题。移除管理目录切换开关，用相册头的折叠能力替代。

**方案 B**（未选）：新增 `albums` 表 + `folders.album_id` + 迁移 + 新 Tauri 命令。为未来多相册留接口。

### 视觉设计

改动前：

```
┌─ 目录 | 相册 ─┐
│ [管理目录 ▼]   │
│   📷 目录1  234│
│   📷 目录2  456│
│   [+ 添加文件夹]│
└───────────────┘
```

改动后：

```
┌─ 目录 | 相册 ─┐
│ [📚 总相册   1234 张 ▼] │    <- 相册头
│   📁 目录1         234  🗑│    <- 缩进 16px
│   📁 目录2         456  🗑│
│   [+ 添加文件夹]        │
└───────────────┘
```

收起后：

```
┌─ 目录 | 相册 ─┐
│ [📚 总相册   1234 张 ▶] │
└───────────────┘
```

### 要点

- **相册头**：图标 `📚`（Lucide `BookImage`）+ 名称"总相册"+ 总照片数（各目录 `photoCount` 之和） + 折叠 chevron
- **目录行**：缩进 16px，图标从 `Image` 改为 `Folder`，右侧照片数 + hover 显示删除按钮
- **管理目录按钮**移除，被相册头折叠替代
- **空状态**：0 个目录时相册头仍保留，下方显示"还没有添加任何目录"

### 状态变化

- 删除 `showManager` 状态
- 新增 `albumExpanded: boolean`（默认 `true`）
- Store 不变

### 交互

| 元素 | 行为 |
|------|------|
| 相册头点击 | 切换 `albumExpanded` |
| 目录行 | 不点击，无主视图过滤 |
| 目录行 🗑 | `confirm` → `removeAlbum`，保留 |
| + 添加文件夹 | 保留 |

### 修改范围

- 只改 `src/components/panel/LeftPanel.tsx`
- 后端、store、api、mock 均不动

4. 空状态相册头仍然显示
